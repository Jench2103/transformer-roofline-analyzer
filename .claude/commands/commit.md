# Commit the Staged Changes

$ARGUMENTS

Please commit the staged changes. The commit message should meet the following requirements.

If a change rationale is provided above, incorporate it into the commit message to explain the motivation behind the changes.

- Ensure the message is concise and clear.
- Describe "why" the changes are made from the concept level.
- List the changes included in the commit.
- Don't describe the iterative workflow of implementation, review, and revision during the work of the commit.
- Follows the message convention shown below.

## Format

```text
<type>(<scope>): <subject>

<description>
```
